The Fragmentation Tax

Liquidity lives across DEXs, fee tiers and L2s. Treasuries burn basis points daily on slippage, latency and manual ops. The winner abstracts venues behind policy-safe, atomic execution and precise telemetry.

Problem Narrative

  • Manual rebalances → governance drag, timing risk, coordination failure.
  • Venue heterogeneity → inconsistent security and UX guarantees.
  • Telemetry gaps → blind spots during volatility spikes.
  • Outcome: persistent capital inefficiency and revenue leakage.

Why Now

  • Institutional flows demand policy guarantees and auditability.
  • L2 maturity enables low-friction automation at scale.
  • Post-cycle sobriety favors durable micro-fee businesses.

Solution — Policy Engine + Adapters

Users/DAO → Policy Engine (rules, limits, oracles) → Adapters (venue connectors) → venues. Guardian pause and timelock guard critical changes. Telemetry makes the system reviewable in real time and governable with confidence.

Policy Engine

  • Allowlists, exposure caps, cooldowns.
  • Oracle staleness (<60s), volatility-aware slippage bands.
  • Velocity limits per asset and per policy.

Adapter Layer

  • Isolated connectors (Uniswap v2/v3, Curve, Balancer, lending).
  • Registry-controlled, hot-swappable with timelock review.
  • Revert reason mapping → actionable on-chain events.

Ops & Governance

  • 3/5 multisig, 48h timelock, guardian pause (no withdrawals).
  • Public dashboards, signed announcements, SIEM webhooks.
  • SLAs and incident runbooks like mission-critical infra.

Security & Assurance

Security is the product. Static guardrails prevent misuse; dynamic checks adapt in flight; ops rigor keeps incidents boring and brief.

Static Guards

  • Reentrancy protection, bounded inputs, allowlists.
  • Deterministic fee accounting; rounding-safe math.
  • Adapter isolation; restricted external call surface.

Dynamic Guards

  • Oracle freshness & volatility-aware slippage bands.
  • Per-asset velocity caps; venue circuit breakers.
  • Fail-shut with explicit on-chain event signaling.

Testing & Monitoring

  • Foundry invariants, fuzzing, mainnet-fork sims; >95% coverage target.
  • Historical exploit replays to validate failure paths.
  • Events: FAILED_TX, CIRCUIT_BREAKER_TRIPPED, POLICY_BREACH_ATTEMPT, FEE_CHANGE_PROPOSED.

Adapter Catalog

Isolated connectors translate policy intents into venue-specific calls. Each adapter increases the strategy surface area without expanding blast radius.

DEX Adapters

  • Uniswap v2/v3: add/remove liquidity, swap within slippage bands, fee tier selection.
  • Curve: pool join/exit with gauge awareness (read-only), metapool constraints.
  • Balancer: weighted pool joins with max price impact control.

Lending Adapters

  • Aave v3: supply/withdraw, E-mode checks, oracle staleness enforcement.
  • Compound v2: mint/redeem with collateral caps per policy.
  • Morpho (readiness): peer-matched supply/withdraw hooks.

Registry & Upgrades

  • On-chain registry approves adapter addresses and versions.
  • Timelock-gated upgrades; diff announcements with commit hash.
  • Canary deploys under stricter policy before generalization.

SDK / API

Two lines to migrate liquidity inside your own dashboard. Intent goes in; policy-safe transactions come out.

TypeScript Example

import { GravitasClient } from '@gravitas/sdk';

const client = new GravitasClient({ signer });

// Migrate $1M USDC under 'treasury-mod' policy
const tx = await client.migrate({
  asset: 'USDC',
  amount: 1_000_000,
  fromVenue: { type: 'UniswapV3', feeTier: 500 },
  toVenue:   { type: 'Curve', pool: '3pool' },
  policyId: 'treasury-mod'
});

await tx.wait(); // <30s on L2

SDK enforces policy locally before calldata is built. Violations never reach chain.

Investor FAQ

Straight answers to common diligence questions.

How do protocol fees accrue?

Fees (5–10 bps) are swept to a protocol treasury controlled by multisig. Initially, funds cover operations and security; future governance can direct surplus toward partner incentives or buybacks.

Is custody of funds ever taken?

No. Partners retain custody. Gravitas manages allowances and executes policy-safe calls into underlying venues via adapters.

Why not build in-house?

Security, audits, adapter maintenance, and ops discipline are expensive to sustain. We specialize so protocols can focus on their core roadmap.

What are the security guarantees?

Static guards (reentrancy, allowlists), dynamic controls (oracle freshness, velocity limits), multisig+timelock, guardian pause (no withdrawals), public telemetry, and external audit before scale.

Token?

Not required for revenue. Considered post-PMF only for governance decentralization and alignment — if and when it adds real value.

What’s the fastest path to value?

Design-partner integration with pre-agreed policy templates, SLAs, and dashboards. We target revenue within 90 days of launch via volume-linked micro-fees.
