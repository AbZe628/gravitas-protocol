Security — Defense in Depth

Security is not a feature; it’s the product. All flows are constrained by immutable checks and executable policies, with change-management visible on-chain.

Static Guards

  • Reentrancy, rate-limits, allowance minimization.
  • Asset & venue whitelists enforced at entry.
  • Invariant checks + revert on policy breach.

Dynamic Guards

  • Oracle freshness & deviation bounds.
  • Adaptive slippage by volatility regime.
  • Velocity caps; circuit breaker events.

Governance-Safe

  • 3/5 multisig with 48h timelock on critical changes.
  • Guardian pause (no withdraw authority).
  • Public telemetry & incident runbooks.

Developer SDK & API

Integrate Gravitas in minutes. Policies configure behavior; the SDK handles execution and receipts.

// TypeScript example
import { GravitasClient } from '@gravitas/sdk';

const client = new GravitasClient(signer);

// Migrate $1M USDC under the 'moderate' policy
const tx = await client.migrate({
  asset: 'USDC',
  amount: 1_000_000,
  policyId: 'moderate',
  fromVenue: 'UniV3-0.05%',
  toVenue: 'Balancer-80/20',
});

await tx.wait(); // <30s typical

SDK Guarantees

  • Typed policies; compile-time safety.
  • Receipts with full event trail.
  • Retry semantics for transient failures.
  • Telemetry hooks out-of-the-box.

Investor FAQ

How do fees accrue?

Micro-fees accrue to a protocol treasury multisig. Early stage: ops & audits. Governance may expand use post-PMF.

Do you take custody?

No. Funds remain in partner-owned contracts or underlying protocols. We operate via allowances within policy bounds.

Token?

Not required. Considered post-PMF for decentralizing governance and aligning long-term incentives.

Why not build in-house?

Security + operations are their own business. Gravitas delivers audited, neutral infra with faster iteration and lower risk.
